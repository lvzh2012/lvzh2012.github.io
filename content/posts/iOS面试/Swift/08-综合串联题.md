# 第八章：综合串联题（高频）

> **学习位置**：前面 1～7 章学完后做本章，把类型系统、派发、闭包、并发、事件链串成完整答案。

## Q1：Swift 调用 `obj.method()` 完整派发过程？

1. 编译器查 method 修饰符
2. **struct/enum / final / private** → 静态派发，直接 call
3. **class 可 override** → VTable 查 IMP
4. **protocol 类型** → Witness Table
5. **@objc dynamic** → `objc_msgSend`（与 OC 相同，见 [OC 第二章](../OC/02-OC动态解析Runtime.md)）
6. 静态派发可被 **WMO（Whole Module Optimization）** 内联

---

## Q2：struct 和 class 怎么选？项目里你怎么用？

**答法模板**：
- 模型、DTO、配置 → `struct`（Codable 友好、线程安全）
- UIViewController、ViewModel 需 OC 互操作 → `class` + `@objc` 或 `NSObject`
- SwiftUI View → `struct`
- 需要 deinit 关资源 → `class` 或 `actor`

---

## Q3：闭包循环引用完整排查思路？

1. 谁持有了闭包？（property、Timer、网络库）
2. 闭包是否 `@escaping`？
3. 闭包内是否捕获 `self`（显式或隐式）？
4. `[weak self]` + `guard let self`
5. 用 Instruments **Leaks / Cycles** 验证

## Q4：async/await 网络请求 + 更新 UI 标准写法？

```swift
@MainActor
final class ProfileVM: ObservableObject {
    @Published var user: User?

    func load() async {
        do {
            user = try await api.fetchUser()
        } catch {
            // handle on main actor
        }
    }
}

// View
.task { await viewModel.load() }
```

**要点**：UI 状态用 `@MainActor`；错误用 `throws` 或 `Result`；避免 `Task { }` 里忘记 weak（VM 通常由 View 持有，生命周期一致）。

---

## Q5：Optional 本质？`if let` 和 `guard let` 区别？

- 本质是 `enum Optional<Wrapped>`
- `if let`：解包成功走分支，适合嵌套
- `guard let`：失败 early exit，后续代码保证非 nil，适合参数校验

---

## Q6：protocol extension 静态派发陷阱？

```swift
protocol Animal { func speak() }
extension Animal { func speak() { print("?") } }
class Dog: Animal { func speak() { print("wang") } }

let d: Animal = Dog()
d.speak()  // wang — 有 requirement 时走 witness table

protocol Plant { }
extension Plant { func grow() { print("grow") } }
struct Tree: Plant { func grow() { print("tree") } }

let t: Plant = Tree()
t.grow()  // grow — 无 requirement，默认实现，非 Tree.grow()
```

**关键**：extension 里 **没有写在 protocol 里的方法** 不能通过 protocol 多态调用具体类型 override。

---

## Q7：Actor 能解决什么问题？和 DispatchQueue 串行队列比？

- **数据竞争**：多 Task 并发读写同一 mutable state
- Actor 保证 **串行 executor** + 编译器 `await` 边界
- 对比串行 GCD：Actor 不阻塞线程，语义更清晰；GCD 仍适合纯 OC 或与 C API 协作

---

## Q8：点击 SwiftUI Button 和 UIKit UIButton 链路异同？

| 阶段 | UIKit | SwiftUI |
|------|-------|---------|
| Hit-Test | 相同 UIKit 树 | Hosting UIView 参与 hitTest |
| 响应 | Target-Action / Gesture | Gesture + 内部回调 |
| 更新 | 手动改 model | `@State`/`@Binding` 触发 body 重算 |

底层触摸仍走 **Hit-Test → 响应链**。

## Q9：Swift 与 OC 混编常见问题？

| 问题 | 解法 |
|------|------|
| Swift 类给 OC 用 | 继承 `NSObject` + `@objc` |
| 协议给 OC 用 | `@objc protocol`（不能有关联类型） |
| 枚举 | `@objc enum` 需 Int 原始值 |
| 闭包 ↔ Block | 自动桥接，注意 `@escaping` |
| 头文件 | `ProductName-Swift.h` |

## Q10：KVO 在 Swift 里怎么用？

```swift
class Person: NSObject {
    @objc dynamic var age: Int = 0
}

let p = Person()
let obs = p.observe(\.age, options: [.new]) { obj, change in
    print(change.newValue ?? 0)
}
// obs 是 KVO token，deinit 时自动移除（iOS 11+）
```

必须 `@objc dynamic` + `NSObject` 子类；底层仍是 Runtime 子类化，见 [OC 第八章 Q2](../OC/08-综合串联题.md)。

---

## Q11：Copy-on-Write 是什么？哪些类型用了？

赋值共享 buffer，写时若引用计数 > 1 则拷贝。`Array`、`Dictionary`、`String`、`Set` 等标准库值类型。面试可提 `isKnownUniquelyReferenced` 做自定义 CoW。

---

## Q12：Sendable 是什么？Swift 6 为什么强调？

标记类型可安全跨并发域传递。Swift 6 默认严格并发检查，非 Sendable 的 class 跨 Task 编译报错。解法：改 struct、用 actor、或 `@unchecked Sendable`（需人工保证）。
