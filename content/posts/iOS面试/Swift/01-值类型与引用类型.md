# 第一章：值类型与引用类型

## 1.1 核心概念

Swift 的类型分为 **值类型（Value Type）** 和 **引用类型（Reference Type）**。

| 类别 | 典型类型 | 存储 | 赋值/传参行为 |
|------|----------|------|---------------|
| **值类型** | `struct`、`enum`、元组、`String`、`Array`、`Dictionary` | 通常栈（大对象可能堆 + CoW） | **拷贝**（Copy-on-Write 优化） |
| **引用类型** | `class`、`closure`（捕获引用时） | 堆 | **共享同一实例**，引用计数管理 |

```swift
struct Point { var x, y: Int }
class Person { var name: String; init(_ n: String) { name = n } }

var p1 = Point(x: 1, y: 2)
var p2 = p1          // 独立拷贝
p2.x = 99
print(p1.x)          // 1

let a = Person("Tom")
let b = a            // 同一对象
b.name = "Jerry"
print(a.name)        // Jerry
```

**面试口诀**：struct/enum 拷贝语义，class 共享语义；Swift 推荐 **优先值类型**。

---

## 1.2 为什么 Swift 推崇值类型？

| 优势 | 说明 |
|------|------|
| **线程安全** | 每个线程持有独立副本，无需锁（小数据） |
| **可预测性** | 修改不影响其他持有者 |
| **无循环引用** | 值类型不涉及 retain cycle |
| **性能** | 栈分配快；CoW 避免不必要拷贝 |

**与 OC 对比**：OC 几乎全是引用类型（对象）；Swift 标准库容器（Array/Dictionary/String）底层是 struct + CoW。

---

## 1.3 Copy-on-Write（CoW）

`Array`、`Dictionary`、`String` 等 **看起来是值类型**，内部用 CoW 优化：

```
赋值时 ──► 共享同一块堆缓冲区（不拷贝）
修改时 ──► 若 buffer 被多处引用 → 先拷贝再改（unique copy）
```

```swift
var a = [1, 2, 3]
var b = a       // 共享 buffer，O(1)
b.append(4)     // 触发 CoW，a 仍为 [1,2,3]
```

**面试点**：CoW 让值类型在大数据场景下也有合理性能；`isKnownUniquelyReferenced` 可手动触发类似优化。

---

## 1.4 struct vs class 选型

| 场景 | 推荐 |
|------|------|
| 纯数据模型（坐标、配置、DTO） | `struct` |
| 需要继承、多态、Objective-C 互操作 | `class` |
| 需要 `deinit` 释放资源 | `class`（struct 无 deinit） |
| 需要 identity（同一性 `===`） | `class` |
| SwiftUI `View` | `struct`（声明式 + 值语义） |

**class 独有**：继承、`deinit`、引用计数、`AnyObject` 约束、KVO（需 `@objc dynamic`）。

---

## 1.5 enum 的 power

Swift 的 `enum` 是 **一等公民**，可带关联值、原始值、方法：

```swift
enum Result<T, E: Error> {
    case success(T)
    case failure(E)
}

enum NetworkError: Error {
    case timeout
    case invalidResponse(code: Int)
}
```

**面试点**：
- `enum` 是值类型，适合建模有限状态
- `switch` 必须 **穷尽**（exhaustive），编译器检查
- `indirect enum` 可递归（如链表、AST）

---

## 1.6 内存布局简答

| 类型 | 布局要点 |
|------|----------|
| **struct** | 按字段顺序排列，可能有 alignment padding |
| **class 实例** | 堆上：metadata 指针 + 引用计数 + 字段 |
| **existential（协议类型）** | 小容器（inline）或大容器（堆 + witness table 指针） |

## 1.7 高频面试 Q&A

### Q1：struct 和 class 的本质区别？

- 赋值：struct 拷贝，class 共享
- 继承：仅 class
- 线程：struct 天然更安全
- 生命周期：class 有 RC + deinit

### Q2：String 是值类型还是引用类型？

**值类型**，内部 CoW；与 OC 的 `NSString`（引用）不同。跨语言桥接时需注意性能（桥接可能触发拷贝）。

### Q3：什么时候 struct 会分配在堆上？

- 被 `class` 持有
- 被 `@escaping` 闭包捕获且逃逸
- 过大或 existential 容器装不下
- 使用 `Any` 擦除后装箱

### Q4：Swift 和 OC 混编时类型怎么选？

- 给 OC 用的 API：`class` + `@objc` + 继承 `NSObject`
- 纯 Swift 模块：优先 struct/enum
