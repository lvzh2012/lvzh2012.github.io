# 第五章：扩展八股（Optional · 协议 · 泛型 · 内存）

> **学习位置**：学完闭包（第三章）和并发（第四章）后阅读。内存管理与 OC [第五章](../OC/05-扩展八股.md) 底层机制相通（ARC、Side Table、weak）。

| 主题 | 要点 |
|------|------|
| **Optional** | 枚举 `.none` / `.some(Wrapped)`，解包方式与安全调用 |
| **ARC** | strong/weak/unowned，与 OC 共用引用计数机制 |
| **协议** | POP、associated type、Self requirement、existential |
| **泛型** | 单态化、where 约束、opaque type `some` |
| **属性包装器** | `@Published`、`@State`、`@AppStorage` 原理 |
| **Error** | `throws`/`try`/`Result`，与 OC `NSError**` 桥接 |
| **访问控制** | open/public/internal/fileprivate/private |
| **Copy 语义** | struct 赋值即拷贝；class 需手动实现 `NSCopying` 或自定义 |

## 5.1 Optional 深度

```swift
enum Optional<Wrapped> {
    case none
    case some(Wrapped)
}
```

| 解包方式 | 安全性 | 示例 |
|----------|--------|------|
| `if let` / `guard let` | 安全 | `if let x = optional { }` |
| `??` 默认值 | 安全 | `optional ?? 0` |
| Optional chaining | 安全 | `user?.address?.city` |
| 强制 `!` | **不安全** | `optional!` 为 nil 时 crash |
| `try?` / `try!` | 错误转 Optional / crash | 网络解析常用 `try?` |

**面试点**：`Optional` 是 enum 值类型；`nil` 在 OC 桥接中是 `NSNull` 的特殊情况需注意。

## 5.2 内存管理（Swift ARC）

| 修饰 | 语义 | 使用 |
|------|------|------|
| `strong`（默认） | 强引用 +1 | 默认所有权 |
| `weak` | 弱引用，不 +1 | delegate、闭包打破循环 |
| `unowned` | 弱引用非可选 | 生命周期确定 |

**与 OC 相同底层**：
- Side Table 管理 weak
- 64 位 non-pointer isa 的 extra_rc
- AutoreleasePool 在 RunLoop 休眠时 drain

**Swift 特有**：
- 值类型无 ARC（除非内含 class 或 CoW buffer）
- `weak`/`unowned` 只能用于 **class 和 class-bound protocol**

## 5.3 协议导向编程（POP）

Swift 推荐 **组合优于继承**：

```swift
protocol Flyable { func fly() }
protocol Swimmable { func swim() }

struct Duck: Flyable, Swimmable {
    func fly() { }
    func swim() { }
}
```

| 概念 | 说明 |
|------|------|
| **Protocol Extension** | 默认实现，非 requirement 的方法有派发陷阱 |
| **Associated Type** | 协议内占位类型，`typealias` 或推断 |
| **Self requirement** | `func clone() -> Self`，existential 上受限 |
| **`some Protocol`** | 不透明类型，编译期固定具体类型 |
| **`any Protocol`** | existential，运行时多态 |

---

## 5.4 泛型

```swift
func swapValues<T>(_ a: inout T, _ b: inout T) {
    let temp = a; a = b; b = temp
}

struct Stack<Element> {
    private var items: [Element] = []
    mutating func push(_ item: Element) { items.append(item) }
}
```

**面试点**：
- 编译期 **单态化**（Specialization），每种具体类型生成一份代码
- `where` 约束：`func f<T>(_ x: T) where T: Equatable`
- 泛型 `<T: P>` vs `any P`：前者性能更好（静态派发）

---

## 5.5 some 与 any

```swift
// some：返回类型固定，调用方不知具体类型但编译器知
func makeShape() -> some Shape { Circle() }

// any：任意 conform 类型
var shapes: [any Shape] = [Circle(), Rectangle()]
```

| | `some P` | `any P` |
|--|----------|---------|
| 类型 | 不透明，固定一种 | existential，多种 |
| 性能 | 更好 | witness table |
| 用于 | 返回值、属性 | 集合存多态 |

## 5.6 属性包装器（Property Wrapper）

```swift
@propertyWrapper
struct Clamped {
    private var value: Int
    var wrappedValue: Int {
        get { value }
        set { value = min(max(newValue, 0), 100) }
    }
    init(wrappedValue: Int) { self.value = wrappedValue }
}

struct Settings {
    @Clamped var volume = 50
}
```

**常见内置**：`@Published`（Combine）、`@State`/`@Binding`（SwiftUI）、`@AppStorage`（UserDefaults）。

**原理**：编译器合成 `_volume: Clamped` 和 `volume` 的 getter/setter。

## 5.7 Error 处理

```swift
enum NetworkError: Error {
    case badURL, timeout
}

func request() throws -> Data {
    guard let url else { throw NetworkError.badURL }
    // ...
}

// 调用
do {
    let data = try request()
} catch NetworkError.timeout {
    retry()
} catch {
    log(error)
}
```

| | Swift | OC |
|--|-------|-----|
| 机制 | `throws` + 枚举 Error | `NSError **` 参数 |
| 桥接 | `NSError` conform `Error` | Swift Error → NSError |

## 5.8 访问控制

| 级别 | 可见范围 |
|------|----------|
| `open` | 模块外可 subclass/override |
| `public` | 模块外可见，不可 subclass |
| `internal` | 模块内（默认） |
| `fileprivate` | 当前文件 |
| `private` | 当前声明域 + 同一 extension |

## 5.9 其他常考点

| 主题 | 一句话 |
|------|--------|
| **defer** | 作用域退出时执行，常用于 unlock/close |
| **lazy var** | 首次访问初始化，非线程安全 |
| **mutating** | struct/enum 修改自身的方法标记 |
| **indirect** | enum 递归 case 间接存储 |
| **KeyPath** | `\Person.name` 类型安全 KVC |
| **Result** | `Result<Success, Failure>` 替代 throws 回调 |
