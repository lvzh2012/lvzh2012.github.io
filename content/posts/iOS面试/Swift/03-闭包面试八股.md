# 第三章：闭包面试八股

> 对应 OC 的 [Block 章节](../OC/03-Block面试八股.md)，概念相通，语法不同。

## 3.1 闭包是什么？

闭包是 **引用类型** 的函数式对象，可以 **捕获上下文变量**。

```swift
var count = 0
let increment = {
    count += 1
}
increment()
print(count)  // 1
```

**与 Block 对比**：

| | OC Block | Swift Closure |
|--|----------|---------------|
| 类型 | C 结构体指针 | 引用类型（堆） |
| 循环引用 | self 强引用 | 默认 strong capture |
| 修饰 | `__block` | `[weak self]` / `[unowned self]` |
| 分类 | Stack/Malloc/Global | escaping / non-escaping |

---

## 3.2 逃逸 vs 非逃逸（必背）

| | `@escaping` | 默认（non-escaping） |
|--|-------------|----------------------|
| 生命周期 | 可超出函数返回后执行 | 必须在函数返回前执行 |
| 捕获 self | 需注意循环引用 | 编译器可优化，较少泄漏 |
| 典型场景 | 网络回调、GCD、存储属性 | `map`/`filter`、同步回调 |

```swift
// 非逃逸：map 的闭包在返回前执行完
let doubled = [1, 2, 3].map { $0 * 2 }

// 逃逸：异步回调
func fetch(completion: @escaping (Data?) -> Void) {
    URLSession.shared.dataTask(with: url) { data, _, _ in
        completion(data)
    }.resume()
}
```

**规则**：闭包作为参数时 **默认 non-escaping**；需要逃逸必须显式 `@escaping`。

---

## 3.3 捕获列表（Capture List）

```swift
class ViewController {
    var name = "VC"

    func setup() {
        // ❌ 循环引用：self → closure → self
        Network.request { data in
            self.handle(data)
        }

        // ✅ weak 打破循环
        Network.request { [weak self] data in
            guard let self else { return }
            self.handle(data)
        }

        // unowned：self 生命周期 ≥ 闭包，否则 crash
        Network.request { [unowned self] data in
            self.handle(data)
        }
    }
}
```

| 捕获方式 | 含义 | 使用场景 |
|----------|------|----------|
| 默认 strong | 强引用 | 闭包生命周期短于 self |
| `[weak self]` | 弱引用，可选 | **最常用**，异步回调 |
| `[unowned self]` | 弱引用，非可选 | self 一定比闭包活得久（如 UIView 动画） |
| `[weak obj = self]` | 重命名捕获 | 避免 guard let self 后命名冲突 |

## 3.4 值捕获 vs 引用捕获

```swift
var x = 10
let closure = { print(x) }
x = 20
closure()  // 20 — 捕获的是引用（class 语义）

// 值捕获：捕获列表里声明
var y = 10
let closure2 = { [y] in print(y) }
y = 20
closure2()  // 10 — 捕获时的快照
```

**面试点**：Swift 闭包对 **引用类型** 捕获的是引用；对 **值类型** 可用 capture list 做拷贝快照。

---

## 3.5 尾随闭包与简化语法

```swift
UIView.animate(withDuration: 0.3) {
    self.view.alpha = 0
}

// 多尾随闭包（Swift 5.3+）
func load(success: () -> Void, failure: (Error) -> Void) { }

load {
    print("ok")
} failure: { error in
    print(error)
}
```

**参数缩写**：`$0`、`$1` 仅适合简单闭包，复杂逻辑降低可读性。

## 3.6 @autoclosure

```swift
func log(_ message: @autoclosure () -> String) {
    #if DEBUG
    print(message())
    #endif
}

log("expensive: \(compute())")  // release 下不执行 compute()
```

**用途**：延迟求值、短路逻辑（`&&`/`||` 的实现）、断言 `assert()`。

**注意**：`@autoclosure @escaping` 可组合，用于存储延迟执行的表达式。

---

## 3.7 循环引用场景汇总

| 场景 | 原因 | 解法 |
|------|------|------|
| self → closure → self | 闭包捕获 self | `[weak self]` |
| 两个 class 互持闭包 | A.closure 捕获 B，B.closure 捕获 A | 一侧 weak |
| Timer / CADisplayLink | target 强引用 | `[weak self]` 或 block-based API |
| Delegate 已是 weak | 闭包内 strong self | 一般安全，注意逃逸闭包存储 |

## 3.8 高频面试 Q&A

### Q1：weak 和 unowned 区别？

| | weak | unowned |
|--|------|---------|
| 类型 | 可选 `T?` | 非可选 `T` |
| 对象释放后 | 自动变 nil | **悬垂引用，访问 crash** |
| 使用 | 通用异步 | self 生命周期确定更长 |

### Q2：为什么 struct 不会有闭包循环引用？

struct 是值类型，闭包捕获 struct 时是拷贝，不涉及引用计数循环。但若 struct 内闭包捕获 **class 实例**，仍可能间接造成 class 的循环引用。

### Q3：@escaping 闭包能否改成 non-escaping？

不能隐式转换。non-escaping 闭包可被编译器优化（如内联），语义不同。

### Q4：GCD 闭包需要 [weak self] 吗？

**推荐总是加**。即使 block 执行完释放，异步期间 self 可能已该 dealloc。详见 [OC 第四章](../OC/04-多线程面试八股.md) GCD 部分。
